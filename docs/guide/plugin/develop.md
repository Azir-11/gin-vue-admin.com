# 插件开发指南

## 标准化插件目录

###  web端口
```

插件名称
  └─web
    └─plugin
      └─插件名称[以上为严格遵循的目录结构]
        ├─api [与后端交互文件夹(可选--由插件功能决定)]
        ├─view  [页面文件夹(可选--由插件功能决定)]
        ├─components  [拆分的组件文件夹(可选--由插件功能决定)]
        ├─utils  [js工具文件夹(可选--由插件功能决定)]
        └─其他  [可自由调整]
```

###  server端
```

后端插件可使用自动插件模板工具生成

插件名称
  └─server
    └─plugin
      └─插件名称 [以上为严格遵循的目录结构]
        ├─api  [对外暴露的路由接收方法文件夹(可选--由插件功能决定)]
        ├─config  [配置文件属性文件夹(可选--由插件功能决定)]
        ├─global  [全局属性文件夹(可选--由插件功能决定)]
        ├─model  [数据模型文件夹(可选--由插件功能决定)]
        │  └─response  [入参模型文件夹(可选--由插件功能决定)]
        ├─router  [路由文件夹(可选--由插件功能决定)]
        ├─service  [具体交互方法文件夹(可选--由插件功能决定)]
        ├─utils  [工具文件夹(可选--由插件功能决定)]
        └─其他  [可自由调整]
```
